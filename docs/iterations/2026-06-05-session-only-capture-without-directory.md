# 未选择保存目录时支持会话截帧

## 背景

用户希望未选择保存目录时仍可截取当前帧，但不写入本地历史或保存目录，只在当前窗口会话内保留记录，刷新后丢失。同时当前窗口会话首次点击“截取当前帧”需要提示这一行为。

## 变更内容

- `App.vue` 将截帧生成和本地持久化拆开：有保存目录时继续写入 PNG、sidecar JSON 和 IndexedDB 历史索引；无保存目录时生成 `storageMode: session` 记录并保留图片 `Blob`。
- 无保存目录时，视频“截取当前帧”不再被拦截，当前窗口第一次点击会提示“只保留在当前窗口会话中，刷新后会丢失”。
- 会话记录支持后续识别、翻译、保存文本、复制图片和下载 PNG，但这些文本更新也只在当前会话有效。
- 历史面板标题从“本地历史”改为“历史记录”，会话记录增加“会话”标签，避免误认为已落盘。

## 接口/兼容性影响

不涉及后端接口变化。前端历史数组新增运行期字段 `storageMode` 和会话记录使用的 `blob`，其中会话记录不会写入 IndexedDB 历史索引。

## 配置/文件影响

不新增配置，不改变保存目录结构。无保存目录时不会创建本地图片文件、sidecar JSON 或历史索引；刷新页面后会话记录自然丢失。

## 验证

- 使用 `@vue/compiler-sfc` 对 `App.vue` 和 `HistoryPanel.vue` 做模板编译检查。
- 未执行完整构建，遵循本项目“小改动优先局部检查”的约定。

## 回滚方式

回滚 `App.vue` 中 `storageMode: session` 相关逻辑、`HistoryPanel.vue` 的标题/标签改动，以及 `styles.css` 的会话标签样式即可恢复原先必须选择保存目录后才能截帧的行为。
