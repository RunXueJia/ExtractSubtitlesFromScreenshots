# 文本按钮改为复制到剪贴板

## 背景

结果面板底部原有“保存文本”按钮会把当前英文字幕和中文翻译写回当前截帧记录。用户希望该按钮改为复制翻译文本到剪贴板，减少导出或保存步骤。

## 变更内容

- `RecognitionResultPanel.vue` 将按钮文案从“保存文本”改为“复制文本”，图标改为文档复制图标，事件名改为 `copy-text`。
- `App.vue` 新增文本剪贴板写入逻辑：优先使用 `navigator.clipboard.writeText`，失败或不可用时使用临时 textarea 后备复制。
- 复制内容只来自当前中文翻译输入框，不包含英文字幕。
- 空翻译时提示“没有可复制的翻译文本。”，复制成功提示“文本已复制到剪贴板”。

## 接口/兼容性影响

不涉及后端接口变化。该按钮不再承担手动保存当前文本到截帧记录的职责；识别和翻译成功时仍会按原流程更新当前记录。

## 配置/文件影响

不新增配置，不改变上传、保存目录或临时文件策略。

## 验证

- 使用 `@vue/compiler-sfc` 对 `App.vue` 和 `RecognitionResultPanel.vue` 做模板编译检查。
- 使用 `rg` 确认前端源码中旧的“保存文本”按钮事件和提示已移除。
- 未执行完整构建，遵循本项目“小改动优先局部检查”的约定。

## 回滚方式

恢复 `RecognitionResultPanel.vue` 中按钮文案、图标和 `save` 事件，并恢复 `App.vue` 中原 `saveCurrentText` 调用 `updateSelectedFrame` 的逻辑即可。
