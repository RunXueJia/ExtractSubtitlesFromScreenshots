# 允许局域网 IP 访问

## 背景

本地开发服务原先绑定 `127.0.0.1`，只能从本机 localhost 访问，不方便在局域网设备上用本机 IP 预览页面。

## 变更内容

- 前端 `npm run dev` 和 `npm run preview` 改为监听 `0.0.0.0`。
- Vite `server` 和 `preview` 配置显式设置 `host: '0.0.0.0'`，端口保持 `5180`。
- 后端新增 `HOST` 配置，默认监听 `0.0.0.0`。
- 后端 CORS 默认放行 localhost、127.0.0.1 和常见局域网私有 IP 来源。

## 接口/兼容性影响

接口路径和响应结构不变。局域网设备可通过 `http://<本机局域网IP>:5180/` 访问前端；直接跨域调用后端时仍可通过 `CORS_ORIGIN` 精确配置来源。

## 配置/文件影响

`.env.example` 新增 `HOST=0.0.0.0`。已有 `server/.env` 不强制修改，未配置 `HOST` 时会使用默认值。

## 验证

已执行前端 Vite 配置加载检查和后端语法检查。

## 回滚方式

将前端脚本和 Vite host 改回 `127.0.0.1`，并将后端 `HOST` 默认值改回仅本机监听即可恢复原行为。
