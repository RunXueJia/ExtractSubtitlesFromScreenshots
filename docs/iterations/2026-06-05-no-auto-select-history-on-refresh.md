# 刷新后不自动选中历史截帧

## 背景

页面刷新并恢复保存目录授权后，会自动选中历史列表第一条记录，导致“截帧预览”默认显示历史截图。用户希望刷新时截帧预览保持空状态，不默认选中历史记录。

## 变更内容

- 调整 `App.vue` 挂载初始化逻辑：历史记录仍从 IndexedDB 加载并在有权限时补齐缩略图，但不再自动调用 `selectFrame(history.value[0])`。
- 历史列表手动点击选择、删除当前项后的自动切换、新截帧后选中当前帧等行为保持不变。

## 接口/兼容性影响

无后端接口变化。前端刷新后的初始选中状态发生变化：`currentFrame` 和 `selectedId` 保持为空，直到用户手动选择历史记录或产生新的截帧。

## 配置/文件影响

无新增配置。无临时文件策略变化。

## 验证

- 已确认触发点为 `App.vue` 挂载阶段自动选择历史第一项。
- 已执行 Vue SFC 编译检查。

## 回滚方式

如需恢复旧行为，可在 `App.vue` 的 `onMounted` 权限成功分支中重新调用 `selectFrame(history.value[0])`。
