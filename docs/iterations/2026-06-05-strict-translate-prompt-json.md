# 严格化字幕翻译模型输出

## 背景

字幕翻译模型可能返回解释、开场白、字段名、Markdown、代码块、格式注释、引号或额外空行，导致前端译文展示混入非译文内容。

## 变更内容

- 将 `TRANSLATE_PROMPT` 当前版本切换为 `v2`。
- 要求模型只返回固定 JSON 对象：`{"text":"中文译文"}`。
- 服务端翻译接口保持对前端返回 `{ text }` 不变。
- 服务端翻译结果复用 JSON `text` 字段解析和保守清洗逻辑，兼容模型偶发输出代码块、前后废话或字段名前缀。
- 将字幕翻译请求的 `temperature` 调整为 `0`，降低输出随机性。

## 接口/兼容性影响

前端接口响应结构不变，仍是：

```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "text": "中文译文"
  }
}
```

模型返回格式的期望从纯文本变为固定 JSON 对象。若模型仍返回旧的纯文本，服务端会保守清洗后继续返回译文。

## 配置/文件影响

无需新增环境变量。未改变上传文件处理、临时文件或外部服务地址配置。

## 验证

- 已进行服务端相关文件语法检查。
- 未运行完整构建，原因是本次为后端提示词与解析逻辑小范围改动。

## 回滚方式

- 将 `server/src/libs/modelPrompts.json` 中 `TRANSLATE_PROMPT.VERSION` 改回 `v1`。
- 如需完全回退解析策略，可恢复 `server/src/libs/modelClient.js` 中 `translateSubtitleToChinese` 直接返回 `callChatCompletion(...)` 的旧逻辑。
